# Contributing Guidelines (Internship)

These rules define **how you should work on this repository**.

## 1. GitHub issues

- Each GitHub Milestone represents **one step of the internship**.
- Each Github Issue represents **one task** within that step.
- Issue titles are numbered: `1.`, `1.1`, `1.2`, `1.11`, etc.
- **Issues must be completed in numerical order**
- Do not skip issues
- Do not start an issue that comes after an unfinished one

## 2. Assigned issues

- Only work on issues **assigned to you**
- Do not self-assign other issues
- Do not modify issue titles or descriptions

## 3. Working in pairs

- You will work **in pairs**
- You must:
  - Perform the task on your own computer
  - Use your own GitHub account
- Helping each other is allowed
- Watching the other person work is **not sufficient**

## 4. Checklist rules

Each issue contains a checklist of targets.

Rules:

- Checkboxes are ticked **only when BOTH of you have completed the task**

---

## 5. Marking an issue as completed

An issue is considered completed **only if all conditions are met**:

1. All checklist items are ticked
2. **Both of you left a comment on the issue**
3. Each intern writes exactly:

```
done
```

## 6. What you must NOT do

- Do not close issues manually
- Do not jump ahead to future issues
- Do not commit directly to `main`

## 7. Purpose

These guidelines ensure that you:
- Learn Git correctly
- Can work independently
- Follow a professional workflow
- Validate real-world development practices
